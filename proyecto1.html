Sobre el proyecto

Caja Negra 𐤟 Devtools 𐤟 JIRA 𐤟 Charles

En este proyecto se realizaron tecnicas para el analisis de pruebas, como la creacion de mapa mental y diagramas de flujo, descomposicion de requerimientos, para una mejor visualizacion y comprension de requisitos.

Para el diseño de las pruebas, se utilizaron tecnicas de pruebas de caja negra. Se realizaron validaciones de datos de entradas con clases de equivalencia y valores limites, se realizaron listas de comprobacion para verificar tanto diseño, comparando el diseño actual de la pagina con los diseños establecidos en Figma, como tambien el funcionamiento de la aplicacion verificando el cumplimiento del documento de requisitos. Aqui tambien se probaron la implementacion de nuevas funciones saltandose el back-end, interceptando las respuestas hacia el navegador mediante brakepoints utilizando Charles Proxy.

Se crearon y ejecutaron casos de pruebas para las principales funciones y se realizaron mas de 50 reportes de errores en Jira.
